## 구현 상세

### 1. mapStore 네비게이션 상태 (`stores/mapStore.ts`)

```ts
interface NavigationState {
  itinerary: Itinerary;
  stopIndex: number;
  isPlaying: boolean;
}
```

- `startNavigation(itinerary)` — 일정의 장소를 마커로 설정하고 첫 경유지로 이동
- `stopNavigation()` — 네비게이션 종료
- `goToStop(index)` — 특정 경유지로 이동 (범위 클램핑)
- `nextStop()` / `prevStop()` — 다음/이전 경유지
- `togglePlayPause()` — 자동 재생 토글

### 2. 3D 경로 렌더링 (`lib/three-scene.ts`)

- **`renderRoute(stops)`** — CatmullRom 곡선 기반 폴리라인 + 글로우 라인
- **`flyToStop(index, stops)`** — 경유지별 45도 앵글 카메라 애니메이션 (cubic ease-in-out, 1.5초)
- **`flyAlongRoute(stops, onStopReached)`** — 순차 비행 (경유지당 ~3.5초 체류)
- **`clearRoute()`** — geometry/material dispose + 애니메이션 취소
- `dispose()`에 `clearRoute()` 추가하여 리소스 누수 방지

### 3. NavigationHUD (`components/map/NavigationHUD.tsx`)

- 상단: 일정 제목 + 닫기 버튼
- 중앙: 현재 경유지 정보 (순번, 도착 시간, 체류 시간, 다음 교통수단)
- 하단: 이전/재생(일시정지)/다음 컨트롤
- 진행률 바 (경유지 기준 %)
- `memo`로 불필요한 리렌더 방지

### 4. ItineraryCard 연결 (`components/chat/ItineraryCard.tsx`)

- 기존 "경로 보기" → "3D 경로 보기"로 변경
- `onClick={() => startNavigation(itinerary)}` 연결

### 5. MapPanel 통합 (`components/map/MapPanel.tsx`)

- `navigation !== null`이면 자동으로 3D 모드 전환
- `NavigationHUD`를 `next/dynamic`으로 lazy-load
- ThreeMap에 `navigation` prop 전달

### 6. ThreeMap 경로 제어 (`components/map/ThreeMap.tsx`)

- 네비게이션 시작 시 `renderRoute()` 호출 (일정 ID 기준 1회만)
- `stopIndex` 변경 시 `flyToStop()` 호출
- `isPlaying` 시 3.5초 간격 자동 `goToStop(index + 1)`
- 네비게이션 종료 시 `clearRoute()` 정리

## 적용된 Best Practices (Vercel React)

| 규칙 | 카테고리 | 적용 내용 |
|------|----------|-----------|
| `bundle-dynamic-imports` | CRITICAL | NavigationHUD를 `next/dynamic`으로 lazy-load — 네비게이션 미사용 시 번들 미포함 |
| `rerender-dependencies` | MEDIUM | effect 의존성을 primitive로 분리 (`navId`, `navStopIndex`, `isNavigating`) |
| `rerender-defer-reads` | MEDIUM | 자동 재생에서 `useMapStore.getState().goToStop()` 사용 — 렌더 시점 구독 제거 |
| `rerender-memo` | MEDIUM | NavigationHUD를 `React.memo`로 래핑 |

## 사용자 플로우

```
채팅에서 "코스 추천해줘" 입력
    ↓
ItineraryCard 렌더링 (타임라인 UI)
    ↓
"3D 경로 보기" 클릭
    ↓
자동 3D 모드 전환 + 경로 폴리라인 렌더링
    ↓
첫 경유지로 카메라 fly (45도 앵글)
    ↓
NavigationHUD 표시
    ├── ◀ 이전 경유지
    ├── ▶ 재생 (자동 비행)
    ├── ▶▶ 다음 경유지
    └── ✕ 네비게이션 종료
```

## 현재 제한사항

- 경로 폴리라인은 직선 보간 (실제 도로 경로 아님) — Phase 2에서 카카오 내비게이션 MCP 연동 시 실제 경로로 대체
- 자동 재생 시 고정 3.5초 간격 — 실제 이동 시간 기반 조절은 미구현
- `LineBasicMaterial`의 `linewidth`는 WebGL 제한으로 일부 GPU에서 1px 고정 — 향후 `Line2` (fat lines)로 교체 가능
