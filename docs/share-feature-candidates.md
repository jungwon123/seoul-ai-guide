# 세션 대화 공유 기능 — 후보 정리

채팅 세션 전체를 외부로 공유하는 기능. 백엔드/프론트 구현 난이도와 UX 를 비교해 우선순위를 결정하기 위한 문서.

---

## 1. 공유 방식 후보

### A. 링크 공유 (서버 저장형)
- **플로우**: 공유 버튼 → 세션 JSON 을 API 로 POST → 짧은 ID 반환 → `/shared/{id}` 링크
- **수신자 경험**: URL 열면 read-only 대화 페이지
- **필요 인프라**: API 엔드포인트 + 스토리지 (DB / KV)
- **장점**
  - 깔끔한 URL, 공유 UX 이상적
  - 만료 정책, 접근 제어 (unlisted/password) 가능
  - 대화 길이 제약 없음
- **단점**
  - 서버 필요 (Vercel KV / Supabase / 자체 백엔드)
  - 비용 (스토리지 / 요청 수)

### B. URL Hash 인코딩 (서버리스)
- **플로우**: 세션 JSON → gzip 또는 lz-string 압축 → base64url → URL hash 에 삽입 (`/share#d=...`)
- **수신자 경험**: 링크 열면 프론트가 hash 디코드해서 렌더
- **필요 인프라**: 없음
- **장점**
  - 즉시 사용 가능, 비용 0
  - 서버 없이 동작
- **단점**
  - 브라우저/서버별 URL 길이 제한 (~2000~8000자) → 긴 대화는 잘림
  - URL 자체에 내용 노출 → 브라우저 히스토리 / 프록시 / 서버 로그에 저장됨
  - 수정 불가, 만료 관리 불가

### C. 마크다운 클립보드 복사
- **플로우**: 공유 버튼 → 대화를 마크다운 포맷으로 변환 → `navigator.clipboard.writeText`
- **수신자 경험**: 유저가 직접 Slack/카톡/메일 등에 붙여넣기
- **필요 인프라**: 없음
- **장점**
  - 매우 단순, 모든 채널에서 유용
  - 수신자가 앱 없이도 읽을 수 있음
- **단점**
  - 장소 카드 / 일정 같은 UI 요소는 텍스트화됨 (클릭 불가)
  - 이미지 없음

### D. 파일 다운로드 (.md / .json)
- **플로우**: 공유 버튼 → Blob 생성 → `a.download` 트리거
- **수신자 경험**: 파일을 받아서 에디터/뷰어로 열람
- **필요 인프라**: 없음
- **장점**
  - 영구 보관, 오프라인 저장
  - JSON 은 추후 import 도 가능
- **단점**
  - 파일 주고받기 자체가 번거로움
  - 공유 흐름 자연스럽지 않음

### E. Web Share API
- **플로우**: 공유 버튼 → `navigator.share({ text, url, title })` → OS 네이티브 공유 시트
- **수신자 경험**: 앱별 포맷 (카톡/메시지/메일/에어드롭)
- **필요 인프라**: 없음. `url` 을 쓰려면 A 또는 B 가 선행돼야 함
- **장점**
  - 모바일 UX 최고, OS 네이티브 통합
  - 유저가 공유 대상 앱 선택
- **단점**
  - 데스크톱 지원 부분적 (Safari 16.4+, Chrome 89+)
  - URL 없으면 text only → UX 제한적

### F. 이미지 내보내기
- **플로우**: `html-to-image` / `dom-to-image` 로 대화 DOM → PNG → 다운로드 or 클립보드
- **수신자 경험**: 이미지 파일 또는 트위터/인스타 업로드
- **필요 인프라**: npm 패키지 1개
- **장점**
  - SNS 공유 친화, 시각적 임팩트
  - 비개발자에게도 이해 쉬움
- **단점**
  - 검색/복사 불가
  - 대화 길면 이미지가 극단적으로 세로로 김
  - 모바일 뷰/데스크톱 뷰 차이로 해상도 문제

## 3. 공유 트리거 위치 (UI)

- **현재 대화**: 상단 헤더 우측에 공유 아이콘 `<Share2 />`
- **과거 세션 (ChatSidebar)**: 각 세션 항목 hover 시 공유 아이콘 노출
- **공유 모달**: 방식 선택 후 실행. 모달 안에서 방식별 버튼 3~5 개

---

## 4. 공유 데이터 범위

포함 대상:
- `messages[]` — 유저 + 에이전트 메시지 텍스트 + 타임스탬프
- `places[]` — 추천 장소 (id, name, address, category, image URL)
- `itinerary` — 일정 (title, stops with placeName / arrivalTime / duration)
- `booking` — 예약 요약 (옵션)

제외 대상:
- 에이전트 종류 / 설정 (개인화 정보)
- 세션 ID / localStorage 키
- 에이전트 세션 컨텍스트 (시스템 프롬프트 등)

---

## 5. 프라이버시 / 보안 고려사항

- 공유 링크는 **unlisted 기본** — URL 아는 사람만 접근
- 만료 기간 옵션 (1일 / 7일 / 30일)
- Password 옵션 (차기)
- 이미지 / 장소 URL 이 외부 CDN 인 경우 저작권 주의
- 공유 전 "개인정보 포함 여부 경고" 토스트 표시

---

## 6. 단계별 로드맵 제안

### Phase 1 — 백엔드 없이 (지금 즉시 가능)
- **C. 마크다운 클립보드 복사** ★ 핵심
- **D. 파일 다운로드 (.md)**
- **E. Web Share API** (모바일 감지 시 버튼 표시)

공유 모달 1개에 위 3개 버튼 배치.

### Phase 2 — 백엔드 추가
- **A. 서버 링크 공유** 추가
- 읽기 전용 `/share/:id` 라우트 + 공유 스키마 저장용 API
- 인프라 후보:
  - Vercel KV (간단, 무료 티어 넉넉)
  - Supabase (Postgres + Row Level Security)
  - 자체 Node/Deno + SQLite

### Phase 3 — 부가 기능
- **F. 이미지 내보내기** (SNS 자랑용)
- **Password 보호** / 열람자 로그
- **수신자가 공유 받은 대화를 자기 세션으로 import**
